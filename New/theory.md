🧠 Теория дня
Тема: "Почему код пишут не только для компьютера"

Самая большая ошибка новичков — думать, что задача программиста написать код, который работает.

Нет.

Задача программиста — написать код, который:

работает;
понятен;
легко изменяется;
легко тестируется;
который через полгода поймёт другой человек.

Есть известная цитата:

"Programs must be written for people to read, and only incidentally for machines to execute."

Запомни её. Она станет одним из наших принципов.

Хороший код

Например:

total_price = quantity * price

Даже без комментариев понятно, что происходит.

Плохой вариант:

x = a * b

Компьютеру всё равно.

Разработчику — нет.

Первое правило нашей команды

Каждое имя должно объяснять своё назначение.

Если переменную приходится комментировать — скорее всего, она плохо названа.

🧠 Теория
Что такое функция?

Представь кофейню.

Клиент говорит:

Сделайте мне латте.

Бариста не придумывает рецепт заново.

Он выполняет уже известную последовательность действий.

То же самое делает функция.

Вместо того чтобы писать один и тот же код несколько раз, мы даём ему имя и вызываем тогда, когда он нужен.

Плохой код
print("Hello Alex")
print("Hello John")
print("Hello Kate")

Если завтра формат приветствия изменится, придётся исправлять каждую строку.

Хороший код
def say_hello(name):
    print(f"Hello, {name}")

Использование:

say_hello("Alex")
say_hello("John")
say_hello("Kate")

Изменить нужно только одну функцию.

Почему это важно?

Запомни правило:

Каждая функция должна делать одну понятную вещь.

Не "обработать всю программу".

А:

добавить книгу;
удалить фильм;
показать меню;
найти пользователя.

Это называется принцип единственной ответственности. Мы ещё не раз к нему вернёмся.

🌐 JavaScript

Сегодня наконец разберём запуск.

Теория

JavaScript можно запускать двумя способами.

1. Через браузер

Используется, когда есть HTML.

Например:

<script src="script.js"></script>

Тогда код выполняется в браузере.

2. Через Node.js

Используется, когда HTML не нужен.

Например:

node script.js

Именно этот вариант мы будем использовать сейчас.
