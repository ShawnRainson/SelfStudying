💻 Блок Dev: Условные операторы if-elif-else и сравнения
Сегодня мы научим программу принимать решения в зависимости от условий.
📖 Теория и правила
В Python для проверки условий используются ключевые слова if (если), elif (сокращение от else if — еще если) и else (иначе).
Знаки сравнения: > (больше), < (меньше), == (равно), != (не равно).
Важно: В Python условия разделяются отступами (4 пробела или клавиша Tab). То, что написано с отступом, выполняется только при истинности условия.
Пример:
python
age = 18
if age >= 18:
    print("Доступ разрешен")
else:
    print("Доступ запрещен")

Вместо того чтобы хранить данные в куче разных переменных, мы можем объединить их в один список (массив), а затем перебрать с помощью цикла.

💻 Блок Dev: Функции (Functions) и ключевое слово return
Функции позволяют упаковать кусок кода, чтобы использовать его многократно, не копируя вручную.
📖 Теория и правила
Функция объявляется с помощью ключевого слова def (от define — определить). Она может принимать параметры (аргументы) и возвращать результат с помощью return.
Пример:
python
def greet_user(username):
    return f"Привет, {username}!"

# Вызов функции:
message = greet_user("Иван")
print(message)
Используйте код с осторожностью.
Важно: Всё, что находится внутри функции, пишется с отступом. return завершает работу функции и отдает результат наружу.

1.	Словарь (dict) — это структура данных, где у каждого значения есть свой уникальный текстовый или числовой ключ (как в реальном словаре: слово — это ключ, определение — значение). Создается в фигурных скобках {}.

python

💻 Блок Dev: Исключения и конструкция try - except
Чтобы твоя программа не «падала» и не закрывалась с кучей непонятных ошибок, когда пользователь ввел что-то не то, в программировании используют обработку исключений.
📖 Теория и правила
Конструкция try - except позволяет «попробовать» выполнить опасный кусок кода. Если в нем произойдет ошибка, программа не сломается, а перейдет в блок except, где мы сможем красиво сказать пользователю, в чем он не прав.
Пример:
python
try:
    number = int(input("Введите число: "))
    print(f"Вы ввели число {number}")
except ValueError:
    print("Ошибка! Вы ввели не число, а текст!")

💻 Блок Dev: Запись в файл (with open)
В Python для работы с файлами используется встроенная функция open(). Но чтобы не забывать закрывать файл (иначе он может повредиться), используют менеджер контекста with. Он автоматически закрывает файл, даже если в коде произойдёт ошибка.
📐 Синтаксис записи:
python
# 'w' означает mode='write' (режим записи). 
# Внимание: режим 'w' полностью перезаписывает файл!
with open('my_tasks.txt', 'w', encoding='utf-8') as file:
    file.write("Hello, World!\n")
    file.write("This is a second line.")
Используйте код с осторожностью.
•	encoding='utf-8' нужен, чтобы корректно записывать русский текст.
•	\n — это оператор переноса строки, чтобы следующая запись шла с новой строчки.

💻 Блок Dev: Чтение из файла (mode='r')
Для чтения файла мы снова используем with open(), но меняем режим 'w' (write) на 'r' (read).
Читать строки из файла можно прямо циклом for — Python сам понимает, что нужно идти строчка за строчкой. При чтении строк из файла в конце каждой строки будет оставаться скрытый символ переноса \n. Чтобы его убрать, к строке применяют метод .strip().
📐 Пример синтаксиса (для понимания):
python
with open('skills.txt', 'r', encoding='utf-8') as file:
    for line in file:
        clean_line = line.strip()  # Убирает невидимый \n и лишние пробелы в начале/конце
        print(clean_line)

with open('tasks.txt', 'r', encoding='utf-8') as file:

Тема дня: Что такое Классы и Объекты?
Сложность: 🚀 Middle-интенсив
Представь, что Класс (Class) — это чертёж автомобиля, а Объект (Object / Instance) — это реальная машина, собранная по этому чертежу. У машины по чертежу есть свойства (цвет, марка) и функции (ехать, тормозить).
В программировании функции внутри класса называются методами, а переменные — атрибутами.
Вот как выглядит простейший класс в Python:
python
class Task:
    # __init__ — это конструктор. Он создаёт свойства объекта при его рождении

🐍 Трек 4: Python
📚 Теория (Инкапсуляция)
В ООП существует концепция инкапсуляции — это сокрытие внутренних данных объекта от прямого вмешательства извне. Представь автомобиль: ты нажимаешь на педаль газа (используешь публичный метод), но не лезешь руками в инжектор, чтобы впрыснуть топливо напрямую.
В Python нет жесткого запрета на уровне языка (как private в C#), поэтому программисты договорились: если переменная внутри класса начинается с одного нижнего подчеркивания (например, self._is_complete), это считается защищенным (protected) атрибутом. Это явный знак для других разработчиков: «Руками извне эту переменную не менять! Используй для этого предназначенные методы!».

🐍 Трек 4: Python (ООП: Геттеры и Сеттеры через @property)
📚 Теория
Инкапсулированные переменные (с нижним подчёркиванием) нельзя трогать напрямую снаружи класса. Чтобы безопасно читать и изменять их, используются свойства. Декоратор @property делает из метода "виртуальную переменную" для чтения, а декоратор .setter позволяет проверять данные перед тем, как записать их в защищённый атрибут.
🔬 Пример синтаксиса (Декораторы свойств)
Пример класса "Банковский аккаунт", где мы защищаем баланс и проверяем, чтобы в него не записали отрицательное число:
python
class BankAccount:
    def __init__(self, money):
        self._balance = money # Защищённый атрибут

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value >= 0:
            self._balance = value
        else:
            print("Ошибка: Баланс не может быть отрицательным!")


# 🐍 Python: ООП Наследование и Алгоритмы (День 17)

### 📚 Теория
Наследование позволяет создать новый класс-потомок, который автоматически заберет все методы и атрибуты класса-родителя, избавляя от дублирования кода. Чтобы вызвать конструктор родительского класса из потомка и передать туда начальные данные, используется встроенная функция `super().__init__()`.

### 🔬 Пример синтаксиса
```python
class Spaceship:
    def __init__(self, name):
        self.name = name

class BattleShip(Spaceship): # Указываем родителя в скобках
    def __init__(self, name, weapons_count):
        super().__init__(name) # Передаем имя родителю
        self.weapons = weapons_count # Уникальное свойство потомка
```

# 🐍 Python: Статические методы @staticmethod (День 18)

### 📚 Теория
Иногда внутри класса нужна функция, которая не работает напрямую с атрибутами конкретного объекта (ей не нужен `self`) и не работает с самим классом (ей не нужен `cls`). Это просто полезная утилита, которая логически связана с темой класса. Такие методы помечают декоратором `@staticmethod`. У них в скобках нет первого обязательного аргумента.

### 🔬 Пример синтаксиса
```python
class Calculator:
    @staticmethod
    def add(a, b): # Нет self, это просто чистая функция внутри класса
        return a + b

# Вызывается напрямую через имя класса, создавать объект НЕ нужно:
result = Calculator.add(5, 10)
```
